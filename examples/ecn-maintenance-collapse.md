# ECN Maintenance Collapse Sample

## Observable Output

- New path is produced and started.
- Execution runs for 2-3 days, then collapses under repeated load.
- Restart attempts recur, but continuity is not sustained.

## Minimal Context

- Time window: 4 consecutive cycles.
- Boundary: one repeated delivery routine.
- Trigger type: load accumulation after initial path launch.

## Chain Trace

- Observable Output: start-stop pattern with repeated collapse points.
- Foreground Entry: task signals enter foreground adequately.
- Historical Template Matching and Explanation: interpretation remains stable.
- New Structure Generation: path generation is successful and specific.
- Execution Maintenance: path continuity fails during repetition load.
- Feedback Update: feedback records collapse points but stability gain is minimal.

## Dominant Breakpoint

- ECN maintenance collapse is the dominant breakpoint.
- The key failure is inability to maintain already-generated path continuity.

## Why Other Explanations Fail

- Not SN over-foregrounding: entry competition does not block startup.
- Not DMN lock-in: interpretation does not force return to old path.
- Not FP failure: new executable path is demonstrably present.

## FCA Interpretation

- Generation succeeds, maintenance fails.
- The chain breaks at sustained load-bearing execution, not at planning or interpretation.

## Update Criterion

- Next cycle must show longer stable run-length under similar load.
- Collapse point should shift later with preserved path integrity.

## Boundary / Reclassification Conditions

- Reclassify to FP generation failure if future cycles no longer produce executable paths.
- Reclassify to feedback non-update if collapse feedback is recorded but never changes maintenance conditions.
