# DMN Template Lock-In Sample

## Observable Output

- Same post-event explanation repeated after each failed attempt.
- New evidence appears, but interpretation remains unchanged.
- Behavioral cycle returns to old response pattern each week.

## Minimal Context

- Time window: 6 weeks.
- Boundary: one recurring collaboration scenario.
- Trigger type: ambiguous partner feedback.

## Chain Trace

- Observable Output: repeated interpretation loop with unchanged action pattern.
- Foreground Entry: relevant social cues enter foreground normally.
- Historical Template Matching and Explanation: old relationship template dominates all interpretation.
- New Structure Generation: candidate alternatives are rejected as non-credible.
- Execution Maintenance: old path remains stable.
- Feedback Update: incoming outcomes are absorbed into old template and do not update next cycle.

## Dominant Breakpoint

- DMN template lock-in is the dominant breakpoint.
- The system preserves continuity by over-weighting prior templates despite contradictory outcomes.

## Why Other Explanations Fail

- Not SN over-foregrounding: entry selection is broad enough.
- Not FP failure alone: alternatives are generated briefly but invalidated by template absorption.
- Not ECN collapse: maintenance is stable, but it maintains the old loop.

## FCA Interpretation

- The chain is locked at historical template interpretation.
- Feedback is processed but structurally neutralized by old-template weighting.

## Update Criterion

- Next cycle must show template-weight shift with different interpretation routing.
- Contradictory outcomes must change path selection, not just narrative wording.

## Boundary / Reclassification Conditions

- Reclassify to feedback non-update if template flexibility appears but outcome linkage still fails.
- Reclassify to FP generation failure if template loosens but no executable new path is produced.
