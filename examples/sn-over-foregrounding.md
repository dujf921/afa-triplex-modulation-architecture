# SN Over-Foregrounding Sample

## Observable Output

- Repeated task-switching every 3-5 minutes.
- Core work never enters sustained execution.
- Output consists of urgent-response fragments only.

## Minimal Context

- Time window: last 10 workdays.
- Boundary: one defined deliverable with stable scope.
- Trigger type: frequent threat-like signals from chat pings.

## Chain Trace

- Observable Output: fragmented responses replace planned output.
- Foreground Entry: incoming threat-like cues repeatedly win priority.
- Historical Template Matching and Explanation: cues are explained as immediate risk.
- New Structure Generation: available but not stably occupied.
- Execution Maintenance: cannot hold because core path never gains foreground window.
- Feedback Update: low; same cue class keeps winning next cycle.

## Dominant Breakpoint

- SN over-foregrounding is the dominant breakpoint.
- The primary failure is entry competition saturation before downstream stages can operate.

## Why Other Explanations Fail

- Not DMN lock-in: explanation is repetitive, but entry saturation appears first.
- Not FP failure: path ideas exist when uninterrupted windows appear.
- Not ECN collapse: no stable execution window is established to maintain.

## FCA Interpretation

- The chain is constrained at foreground entry.
- Downstream stages are secondarily compressed by repeated high-priority threat capture.

## Update Criterion

- Next cycle shows stable foreground windows for core task blocks.
- Core path enters sustained execution before reactive fragments dominate.

## Boundary / Reclassification Conditions

- Reclassify to FP generation failure if foreground windows exist but no new path forms.
- Reclassify to ECN maintenance collapse if path exists and starts, but repeatedly breaks under load.
